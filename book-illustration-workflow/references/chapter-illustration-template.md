# 章节截图总表模板

在开始做某一章截图前，先复制这份表，逐项填完。这样后面的截图执行、原文回填和 Feishu 同步都不会乱。

## 用法

1. 先通读章节
2. 把所有图号列出来
3. 区分哪些是：
   - 原始数据图
   - Claude Code 对话图
   - 成品图
   - 对比图
   - 官方页面图
4. 给每张图确定：
   - 真实提示词
   - 截图对象
   - 文件名
   - 原文位置
5. 每完成一张就更新状态

## 提示词模板

### 1. 安装图

```text
/plugin marketplace add anthropics/skills
```

```text
/plugin install document-skills@anthropic-agent-skills
```

### 2. 检查图

```text
请先检查“<数据文件夹>”里的所有表格，告诉我目前有哪些明显问题，重点看：

1. 表头是否一致
2. 日期格式是否统一
3. 是否有空值
4. 是否有重复记录
5. 哪些字段可能需要我确认后再处理

先给我问题清单，不要直接修改文件。
```

### 3. 正式处理图

```text
请处理“<数据文件夹>”里的所有表格，并按下面规则输出最终汇总表：

1. 合并多份同类表格
2. 把“<字段别名A>”“<字段别名B>”“<字段别名C>”统一成“<统一字段名>”
3. 日期统一成 YYYY-MM-DD
4. 删除完全重复的记录
5. 对缺失值按规则标记
6. 保留核心字段
7. 输出一份整理后的汇总表
8. 另外给我一份处理说明
```

### 4. 图表图

```text
请根据“<汇总文件.xlsx>”生成一张适合汇报使用的<趋势图/对比图>，并导出为 PNG 图片，要求：

1. 主题明确
2. 让人一眼看出重点
3. 风格简洁、专业
4. 图片尺寸适合直接用于PPT或文档
```

### 5. 报表图

```text
请根据“<汇总文件.xlsx>”生成一页适合直接用于周报或月报的报表页面，并导出为 PNG 图片，要求：

1. 包含核心指标
2. 包含至少一张图表
3. 版面完整
4. 风格简洁、专业
```

## 回填检查表

```md
| 检查项 | 是否完成 |
| --- | --- |
| 每张图都已经落到正确图号 | [ ] |
| 文件名和图号完全一致 | [ ] |
| 正文里没有作者备注 | [ ] |
| 图片位置跟着正文语义走 | [ ] |
| Feishu 文档顺序和本地一致 | [ ] |
| 不存在错图或重复图 | [ ] |
```
