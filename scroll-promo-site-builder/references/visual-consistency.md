# Visual consistency

## Lock order

1. Brand character or brand-object identity.
2. Core vehicle, machine, environment, or delivery object.
3. Palette and material system.
4. Lighting and camera system.
5. Scene composition and copy safe area.

## Identity sheet requirements

Record invariant features that must survive every generation:

- Silhouette.
- Face or front-view geometry.
- Primary and accent colors.
- Clothing, accessories, or brand marks.
- Relative scale.
- Signature gestures.
- Prohibited mutations.

Generate front, three-quarter, side, and action views when the model and composition allow it.

## Device or world sheet requirements

Record:

- Exterior shape and proportions.
- Functional zones.
- Input and output positions.
- Movement direction.
- Materials and wear level.
- Repeated symbols or colors.
- Camera-height reference.

## Text strategy

- Use HTML or post-production overlays for exact titles, labels, counts, and CTAs.
- When an object must be recognized in the generated scene, combine a large blank plaque, distinct icon, packaging shape, and context.
- If generated object text is necessary, keep it short, large, front-facing, and high contrast; validate every character.
- Never trust small generated UI text.

## Reference discipline

- Use the approved identity sheet in every relevant image2.0 call.
- Use approved scene frames as Seedance references.
- Do not ask later generations to reconstruct identity from prose alone.
- Preserve one direction of travel across adjacent scenes unless reversal is part of the story.
